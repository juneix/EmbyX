# 修复 Android APK 构建失败

## 问题

Run `27925086397`（"安卓 APK 签名" commit `fddbf96`）构建失败，en/zh 两个变体都在 `Build Debug APK` 步骤报错：

```
FAILURE: Build failed with an exception.
* What went wrong:
Could not find method signingConfig() for arguments [SigningConfig$AgpDecorated_Decorated{name=release, ...}]
```

## 根因

`.github/patch_android.py` 第 219-222 行的 `str.replace("release {", ...)` **过于宽泛**。

脚本分两步注入签名：
1. 先插入 `signingConfigs { release { ... } }` 块（第 206-217 行）
2. 再对 `buildTypes` 内的 `release` 和 `debug` 注入 `signingConfig signingConfigs.release`（第 219-222 行）

第二步的 `gradle.replace("release {", ...)` 匹配了**所有** `"release {"` 出现，包括刚插入的 `signingConfigs { release {` 块，导致在 signing config 定义内部多插入了一行 `signingConfig signingConfigs.release`，Groovy 语法错误。

## 修复方案

**文件**: `.github/patch_android.py`

修改第 219-222 行，用正则精确匹配 `buildTypes` 作用域内的 `release {`，避免误匹配 `signingConfigs` 内的同名块：

```python
if "signingConfig signingConfigs.release" not in gradle:
    gradle = re.sub(
        r'(buildTypes\s*\{[^}]*?)release\s*\{',
        r'\1release {\n            signingConfig signingConfigs.release',
        gradle,
        count=1
    )
```

仅修改 `release {` 的 replace 逻辑，`debug` 的注入方式不受影响（`buildTypes {` 字符串在文件中唯一）。

## 验证

1. 本地模拟：读取 Capacitor 生成的 build.gradle 模板，运行 patch 脚本，检查输出的 build.gradle 语法正确
2. 推送后触发 CI，确认 en/zh APK 构建通过
